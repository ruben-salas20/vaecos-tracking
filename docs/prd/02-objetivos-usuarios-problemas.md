# 02. Objetivos, Usuarios y Problemas

## Objetivos principales

1. Reducir trabajo manual operativo.
2. Eliminar dependencia progresiva de Notion.
3. Asegurar trazabilidad completa y auditable.
4. Mejorar visibilidad con analytics accionables.
5. Migrar a una aplicacion web cloud always-on.

## Prioridad declarada

En la etapa actual se prioriza:

- Reducir trabajo manual.
- Trazabilidad.
- Analytics.

## No objetivos inmediatos

- SaaS comercial en esta fase.
- Integraciones externas no criticas antes de estabilizar core.
- Ampliar carriers sin estabilizar flujo actual.

## Usuarios

### Primarios

- Operadores de tracking (4 usuarios).

### Secundarios (proxima fase)

- Supervisor operativo.
- Admin tecnico/operativo.

## Problemas a resolver

1. Alta carga manual para revision de excepciones.
2. Dependencia operativa de Notion.
3. Falta de colaboracion multiusuario en una sola fuente de verdad.
4. Necesidad de trazabilidad robusta.
5. Flujo operativo sin estandar completo de ejecucion-resolucion-mejora.
